Formas como as empresas podem usar a API da Getty Images

Gere um portal na Internet que publica histórias sobre avistamentos de celebridades. Pretende pesquisar e apresentar as fotografias mais recentes de personalidades na sua aplicação.

Estes procedimentos, utilizando a nossa ferramenta API baseada na Internet denominada Swagger, ajudarão a orientá-lo passo a passo ao longo do processo de iniciar uma chamada da API da Getty Images que irá solicitar fotografias à plataforma da Getty Images para apresentação na sua aplicação.

Este artigo contém cenários de fluxos de trabalho que fornecem informações sobre as formas como as empresas podem usar a API da Getty Images.

  • Pesquisar por personalidades e criar uma galeria de imagens
  • Pesquisar por imagens incorporáveis de personalidades e ver os resultados da pesquisa

Antes de começar

Certifique-se de que tem uma chave da API.

Pesquise por imagens de personalidades e veja os resultados da pesquisa

  1. Abra a Swagger.
  2. Na coluna da esquerda, clique em Pesquisa e, em seguida, clique em GET /v3/search/images/editorial.

  3. Na caixa da chave da API, introduza a sua chave da API de área de segurança.

    4. Como posso obter uma chave da API de área de segurança?

  4. No painel campos, selecione uma ou mais opções de metadados (por exemplo, legenda, imagem para maqueta, título e artista) para serem apresentadas com os resultados da pesquisa.

    Campo

    Descrição

    allowed_use

    Comunica informações sobre licenças para clientes que realizam compras de imagens individuais. A maioria dos clientes da API acede a imagens através dos seus contratos e não precisa de consultar este campo.

    artist

    Identifica o criador do ficheiro.

    asset_family

    Especifica o filtro de família do ficheiro a aplicar aos resultados da pesquisa. Os valores possíveis são:

    • Creative
    • Editorial

    call_for_image

    Indica que tem de contactar o seu representante de vendas da Getty Images para obter acesso a ficheiros de alta resolução para esta imagem. Esta imagem não está disponível para download online.

    caption

    Uma breve descrição do ficheiro.

    collection_code

    Um código de três letras que identifica a coleção.

    collection_id

    Identifica a coleção à qual pertence o ficheiro.

    collection_name

    Identifica o nome da coleção à qual pertence o ficheiro.

    color_type

    Indica se o ficheiro é a cores ou a preto e branco. Os valores possíveis são:

    • Color
    • Black/White

    comp

    Dimensão de visualização grande; dimensionado para composição em layout. As imagens para maqueta são maiores do que as de pré-visualização e miniaturas, mas não são de tamanho completo, numa média de cerca de 600 píxeis no lado mais comprido.

    copyright

    Identifica direitos de autor do ficheiro.

    date_created

    Identifica a data de criação do ficheiro.

    detail_set

    Uma coleção de campos de metadados habitualmente usados para apresentar informação numa experiência de pesquisa baseada em resultados.

    editorial_segments

    Indica a categoria editorial à qual o ficheiro pertence, se a imagem estiver na família de imagens editoriais. Os valores possíveis são:

    • News
    • Sports
    • Entertainment
    • Fashion
    • Celebrity portraiture
    • Royals
    • Premium photojournalism
    • Video

    event_ids

    Especifica a ID de um evento associado ao ficheiro. Apenas relevante para imagens de categoria editorial.

    graphical_style

    Indica o estilo gráfico do ficheiro. Os valores possíveis são:

    • Photograph
    • Illustration

    id

    Uma cadeia única de carateres numéricos que identifica o ficheiro.

    keywords

    Identifica o tipo de palavra-chave. Os valores possíveis são:

    • Unknown
    • Specific people
    • Ethnicity
    • Age
    • Gender
    • Number of people
    • Concept
    • Location
    • Viewpoint
    • Image technique
    • Composition
    • Color
    • Additional info
    • Candidate term
    • Fashion phrases
    • Entertainment

    largest_downloads

    Devolve resultados de links de download hipermídia para o tamanho maior.

    license_model

    Identifica um modelo de licença para o ficheiro. Os valores possíveis são:

    • RoyaltyFree
    • RightsManaged

    max_dimensions

    As dimensões em píxeis do maior ficheiro disponível.

    orientation

    Especifica a orientação de ficheiros devolvidos em resultados de pesquisa. Os valores possíveis são:

    • Horizontal
    • Vertical
    • PanoramicHorizontal
    • PanoramicVertical
    • Square

    people

    Especifíca os nomes de pessoas em ficheiros devolvidos no resultado de pesquisa.

    preview

    Tamanho de visualização médio; dimensionado para pré-visualizações pop-up. Imagens de pré-visualização são maiores do que miniaturas mas menores do que imagens para maqueta, e são de cerca de 270 píxeis no lado mais comprido.

    product_types

    Identifica a oferta de produto a um cliente que autoriza acesso a um ficheiro. Os valores possíveis são:

    • easyaccess
    • editorialsubscription
    • premiumaccess
    • royaltyfreesubscription
    • imagepack

    referral_destinations

    Contém uma entrada ReferralDestination para cada um dos websites nos quais o ficheiro pode ser encontrado.

    summary_set

    Uma coleção de campos de metadados habitualmente usados para apresentar informação numa experiência de pesquisa baseada em resultados.

    thumb

    Tamanho de visualização menor; dimensionado para uma grelha de resultados. As imagens de miniatura são menores do que as de pré-visualização e as imagens para maqueta, e têm uma dimensão de 170 píxeis no lado mais comprido.

    title

    Indica o título do ficheiro.

    uri_oembed

    Identifica um URI que pode ser usado para obter dados oEmbed para o ficheiro. Lembre-se que este campo apenas será preenchido para ficheiros incorporáveis. Consulte a nossa documentação oEmbed para obter mais informações.

  5. Na caixa ​ordem de classificação, selecione ​most_popular.

    Ordenar os resultados da pesquisa ajuda-o a visualizar, a organizar e a tomar decisões rápidas e eficazes sobre fotografias e vídeos que está a pesquisar. Best_match ordena os resultados por imagens recentes e relevantes para a sua pesquisa; most_popular ordena resultados de acordo com as imagens mais populares pesquisadas pelos clientes ao longo do tempo; newest ordena os resultados de acordo com a data de upload mais recente; oldest ordena os resultados de acordo com a data de upload mais antiga.

  6. Na caixa da frase, digite Taylor Swift.

    Nota:: Digitar uma frase na caixa da frase realiza a mesma função do que digitar uma frase na caixa de pesquisa no website gettyimages.com.

    Pretende resultados de pesquisa referentes a uma personalidade diferente? Basta introduzir um nome diferente nesta caixa.

  7. Clique no botão Experimentar.

    A caixa Corpo de Resposta lista cada imagem de resultados de pesquisa e os campos de metadados que selecionou no passo 4.

  8. Para ver uma imagem dos seus resultados de pesquisa, no Corpo da resposta, copie o endereço HTTP completo (não incluir as aspas) apresentado no campo uri a seguir, e cole-o na barra de endereço do seu browser predefinido.

    Neste exemplo, o endereço HTTP completo a copiar (não incluir as aspas) é:

    http://cache2.asset-cache.net/gc/488755785-charles-james-beyond-fashion-costume-gettyimages.jpg?v=1&c=IWSAsset&k=2&d=X7WJLa88Cweo9HktRLaNXmrPtuUNDC4UM%2f%2bnurWTaUiLHfjnKSTc9Er6OUTQtSUrO1LhqjmtqFmEtYaCxpt12Q%3d%3d&b=ODU2

    O endereço HTTP apresenta a seguinte imagem:

    A caixa de URL Solicitado mostra o código específico que uma aplicação terá de enviar para a API para poder obter resultados de pesquisa como estes. Repare como o pedido muda quando seleciona diferentes frases e opções na Swagger. Enviar este código para a API faz com que uma resposta seja devolvida à aplicação.

    A caixa Corpo de Resposta mostra como esta resposta será apresentada para diferentes tipos de pedidos. O código da resposta contém toda a informação de que um programador necessita para poder criar uma aplicação que apresente a imagem e a informação relativa à mesma.

Pesquisar por imagens incorporáveis de personalidades e ver os resultados da pesquisa

  1. Abra a Swagger.
  2. Na coluna da esquerda, clique em Pesquisa e, em seguida, clique em GET /v3/search/images/editorial.

  3. Na caixa da chave da API, introduza a sua chave da API incorporável.

    4. Como posso obter uma chave da API incorporável?

  4. No painel campos, selecione uri_oembed.

    Este campo de metadados irá fornecer o URI que pode ser usado para incorporar uma imagem no seu website.

    Sugestão: Também pode selecionar uma ou mais opções de metadados (por exemplo, legenda, imagem para maqueta, título e artista) para serem apresentadas com os resultados da pesquisa.
