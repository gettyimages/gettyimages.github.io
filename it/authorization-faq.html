FAQ: Utilizzo di OAuth 2.0 per accedere agli API Getty Images

Gli API Getty Images richiedono l'utilizzo del protocollo OAuth 2.0 per il recupero dei token di autorizzazione e autenticazione. Questi token sono utilizzati per:

  • identificare chi sei (autenticazione)
  • identificare a quale contenuto hai accesso o per quale contenuto hai una licenza (autorizzazione)

Sebbene sia possibile effettuare alcune chiamate API con solamente una chiave API (ad esempio, gli endpoint di ricerca /v3/search/images e /v3/images non richiedono un token di autorizzazione), non lo consigliamo perché la risposta non è del tutto accurata. Ad esempio, le richieste di ricerca effettuate con solo una chiave restituiranno materiali che non potrai utilizzare perché tali materiali non rientrano nell'ambito della licenza concessa per il tuo utilizzo.

Domande frequenti sull'autorizzazione

D: come mi assicuro che i risultati della mia ricerca contengano solo immagini o video per il cui uso ho una licenza?

R: includi sempre un token nell'intestazione della tua ricerca, richiesta di immagini e video. Il token identifica chi sei e per quali materiali hai una licenza da utilizzare in modo che il motore di ricerca riporti solo materiali da quella collezione di contenuti.

D: quando effettuo una richiesta di download, ottengo l'errore "non autorizzato". Perché?

R: un motivo per il quale potresti ottenere tale errore è perché l'immagine o il video che stai tentando di scaricare non rientra nell'ambito del tuo contratto di licenza. Un altro motivo potrebbe essere la necessità di un token per tutti i download richiesti e la tua richiesta potrebbe non avere un token nell'intestazione dell'autorizzazione HTTP.

D: un token ha una scadenza?

R: sì. I token generati da Client Credentials e Resource Owner hanno una durata di 30 minuti.

D: come consento agli utenti di accedere con il loro nome utente Getty Images?

R: Il flusso OAuth 2.0 Implicit Grant è stato implementato per questo scopo. Consulta Autorizzazione flussi di lavoro per ulteriori informazioni su Implicit Grant.

Autorizzazione flussi di lavoro

Getty Images API utilizza il protocollo OAuth per concedere l'accesso alle applicazioni client per la funzionalità API. La tua applicazione client utilizzerà OAuth 2.0 per richiedere un token di accesso dal Server di Autorizzazione Getty Images, estrarre un token dalla risposta e successivamente inviare il token a Getty Images API per richiedere l'accesso a funzioni come ricerca e download di immagini. Getty Images API supporta i seguenti tre flussi OAuth 2.0:

  • Client Credentials
  • Resource Owner
  • Implicit Grant

Flussi OAuth 2.0 supportati da Getty Images API

Client Credentials

Il flusso Client Credentials è per applicazioni client che non hanno come utenti individui singoli. Un'applicazione Sandbox, intesa per uno sviluppo di prova e che non richiede un contratto di licenza, può utilizzare solamente il flusso Client Credentials.

Se la tua azienda ha un contratto di licenza con Getty Images che concede accesso a specifiche immagini, l'utilizzo del flusso Client Credentials per richieste di ricerca ti darà dei risultati poco accurati rispetto ai contenuti a cui hai accesso e non sarai in grado di scaricare i file. Per ottenere risultati di ricerca e scaricare materiali che rientrano nell'ambito del tuo contratto di licenza, la tua chiave deve essere configurata come token pienamente autorizzato quando si usa Client Credentials. Per saperne di più su una particolare configurazione, contatta l'account manager Getty Images.

Il flusso Client Credentials concede un token per un accesso di 30 minuti. Una volta che il token è scaduto, devi richiedere un altro token. La maggior parte delle applicazioni recuperano un nuovo token prima che quello attuale scada.

Ottenere il token Client Credentials
  1. Apri lo strumento web API Getty Images.
  2. Nella colonna di sinistra, clicca su OAuth2 e poi su POST /oauth2/token – Client Credentials.
  3. Nella casella client_id, inserisci la tua chiave di sicurezza API.

    Come ottengo una chiave di sicurezza API?

  4. Nella casella client_secret, inserisci il tuo API segreto.

    Dove trovo il mio API segreto?

    Apri il sito Getty Images API e nella barra di navigazione in alto a destra, clicca su Accedi. Inserisci il tuo nome utente e la password, clicca su Accedi, e poi accedi alla pagina Account personale.

  5. Clicca su Ottieni il Token.

    Note

    • Utilizza questo token nelle tue chiamate agli endpoint API.
    • Il flusso Client Credentials concede un token per un accesso di 30 minuti. Una volta che il token è scaduto, devi richiedere un altro token.

Ottenimento del token Resource Owner Grant
  1. Apri lo strumento web API Getty Images.
  2. Nella colonna di sinistra, fai clic su OAuth2 e poi clic su POST /oauth2/token – Resource Owner Grant.

  3. Nella casella client_id, inserisci la tua chiave API.

    4. Come ottengo una chiave di sicurezza API?

  4. Nella casella client_secret, inserisci il tuo API segreto.

    Dove trovo il mio API segreto?

    Apri il sito Getty Images API e nella barra di navigazione in alto a destra, clicca su Accedi. Inserisci il tuo nome utente e la password, clicca su Accedi, e poi accedi alla pagina Account personale.

  5. Nella casella nome utente, inserisci il tuo nome utente.
  6. Nella casella password, inserisci la tua password utente.
  7. Clicca su Ottieni il Token.
Ottenimento del token di aggiornamento
  1. Apri lo strumento web API Getty Images.
  2. Nella colonna di sinistra, clicca su OAuth2 e poi su POST /oauth2/token – Refresh Token.

  3. Nella casella client_id, inserisci la tua chiave API.

    4. Come ottengo una chiave di sicurezza API?

  4. Nella casella client_secret, inserisci il tuo API segreto.

    Dove trovo il mio API segreto?

    Apri il sito Getty Images API e nella barra di navigazione in alto a destra, clicca su Accedi. Inserisci il tuo nome utente e la password, clicca su Accedi, e poi accedi alla pagina Account personale.

  5. Nella casella refresh_token, inserisci il valore delrefresh_token che hai ricevuto in risposta alla tua richiesta iniziale di token Resource owner.
Implicit Grant

In tale flusso, l'applicazione client permette agli utenti di accedere con il loro nome utente e password Getty Images. Comunemente utilizzato dalle applicazioni i cui utenti sono anche clienti di Getty Images (CMS, DAM, plug-in, e così via), questo flusso consente all'applicazione client di riportare risultati di ricerca rilevanti di materiali che rientrano nell'ambito del contratto di licenza Getty Images concesso agli utenti.

Gli utenti vengono indirizzati alla pagina di accesso Getty Images per inserire un nome utente e una password e poi reindirizzati all'applicazione. Il token generato tramite questo flusso è valido per un anno.

Una volta che un token di accesso è scaduto per una determinata risorsa, un nuovo token di accesso deve essere recuperato per accedere a tale risorsa. Il flusso Implicit Grant non supporta l'aggiornamento del token d'accesso. Nuovi token di accesso devono essere recuperati attraverso il flusso Implicit Grant. Notare che se la password viene modificata, il token non sarà più valido.

  1. L'applicazione client chiama l'endpoint OAuth2 (ad esempio, https://api.gettyimages.com/oauth2/auth/) e trasmette le seguenti informazioni:
    • Chiave API / ID Cliente
    • Reindirizzamento URI che è stato registrato con Getty Images API (potrebbero essere aggiunti parametri non registrati)
    • Tipo di token di risposta
    • Stato (facoltativo)
  2. L'applicazione client reindirizza alla pagina di accesso Getty Images la cui posizione viene fornita nella risposta al punto 1.
  3. L'utente finale accede con il proprio nome utente o password Getty Images o Thinkstock, e poi clicca Autorizza.
  4. L'API verifica le credenziali del cliente e dell'utente e successivamente reindirizza l'applicazione client con un token di accesso a lunga durata.
