Comment les entreprises peuvent utiliser l'API de Getty Images

Vous gérez un portail Internet qui publie des infos sur les célébrités. Vous voulez chercher et afficher les dernières photos de stars dans votre application.

Ces procédures, utilisant notre outil API Swagger, vous guideront pendant l'initialisation d'un appel API de Getty Images. Cet appel est chargé d'effectuer la demande de photos sur la plate-forme Getty Images pour les afficher dans votre application.

Cet article contient des scénarios de flux de travail pour vous donner des conseils sur la manière dont une entreprise peut utiliser l'API de Getty Images.

  • Rechercher des stars et créer une galerie d'images
  • Rechercher des images de stars à incorporer et visualiser les résultats de recherche

Avant de commencer

Assurez-vous de disposer d'une Clé API.

À l'attention des développeurs qui créent des applications avec l'API de Getty Images : Pour vous assurer que les résultats de votre recherche vous renvoient aux produits auxquels votre application peut accéder, vous devez, en plus de votre clé API, obtenir un jeton d’accès à utiliser avec ces flux de travail.

Rechercher des images de stars et visualiser les résultats de recherche

  1. Ouvrez Swagger.
  2. Dans la colonne de gauche, cliquez sur Rechercher puis sur GET /v3/search/images/editorial.

  3. Dans le champ Clé-Api, entrez votre clé Api Sandbox.

    4. Comment obtenir une clé Api Sandbox ?

  4. Dans le panneau des champs, sélectionnez au moins une option de métadonnées (par exemple, légende, maquette, titre et artiste) à afficher avec les résultats de recherche.

    Champ

    Description

    allowed_use

    Communique les informations sur la licence aux clients qui réalisent des achats d'image ponctuels. La plupart des clients API ont accès aux images au travers de leurs contrats et n'ont pas besoin de consulter ce champ.

    artist

    Identifie le créateur du produit.

    asset_family

    Spécifie le filtre de la famille de produit à appliquer aux résultats de la recherche. Les valeurs possibles sont :

    • Creative
    • Editorial

    call_for_image

    Indique que vous devez appeler votre responsable de clientèle Getty Images pour obtenir l'accès à des fichiers haute résolution pour cette image. Cette image n'est pas disponible pour un téléchargement en ligne.

    caption

    Une brève description du produit.

    collection_code

    Un code de trois lettres identifiant la collection.

    collection_id

    Identifie la collection à laquelle appartient le produit.

    collection_name

    Identifie le nom de la collection à laquelle appartient le produit.

    color_type

    Indique si le produit est en couleur ou en noir et blanc. Les valeurs possibles sont :

    • Color
    • Black/White

    comp

    Taille grand écran ; taille de composition de maquette. Les images de maquette sont plus grandes que les aperçus et les vignettes, mais ne sont pas de taille maximale, de 600 pixels en moyenne sur le côté le plus long.

    copyright

    Identifie les droits d'auteur du produit.

    date_created

    Identifie la date de création du produit.

    detail_set

    Une collection de champs de métadonnées communément utilisée pour afficher les informations lors d'une expérience basée sur une recherche de résultats.

    editorial_segments

    Indique la catégorie éditoriale à laquelle appartient le produit, si l'image fait partie de la famille d'image Photos d'actualités. Les valeurs possibles sont :

    • News
    • Sports
    • Entertainment
    • Fashion
    • Celebrity portraiture
    • Royals
    • Premium photojournalism
    • Video

    event_ids

    Spécifie l'ID d'un événement associé avec le produit. Pertinent uniquement pour les Photos d'actualités.

    graphical_style

    Indique le style graphique du produit. Les valeurs possibles sont :

    • Photograph
    • Illustration

    id

    Une chaîne numérique unique qui identifie le produit.

    keywords

    Identifie le type de mot-clé. Les valeurs possibles sont :

    • Unknown
    • Specific people
    • Ethnicity
    • Age
    • Gender
    • Number of people
    • Concept
    • Location
    • Viewpoint
    • Image technique
    • Composition
    • Color
    • Additional info
    • Candidate term
    • Fashion phrases
    • Entertainment

    largest_downloads

    Renvoie les liens de téléchargement hypermédia à la taille la plus grande.

    license_model

    Identifie un modèle d'octroi de licence pour le produit. Les valeurs possibles sont :

    • RoyaltyFree
    • RightsManaged

    max_dimensions

    Les dimensions en pixels du plus grand fichier disponible.

    orientation

    Spécifie l'orientation de produits correspondants dans des résultats de recherche. Les valeurs possibles sont :

    • Horizontal
    • Vertical
    • PanoramicHorizontal
    • PanoramicVertical
    • Square

    people

    Spécifie les noms des personnes dans les produits correspondants dans des résultats de recherche.

    preview

    Taille affichage moyen ; mis à la taille pour aperçus contextuels. Les aperçus sont plus grands que les vignettes, mais plus petits que les maquettes, de 270 pixels en moyenne sur le côté le plus long.

    product_types

    Identifie l'offre de produits pour un client autorisant l'accès à un produit. Les valeurs possibles sont :

    • easyaccess
    • editorialsubscription
    • premiumaccess
    • royaltyfreesubscription
    • imagepack

    referral_destinations

    Contient une entrée Recommandation/Destination pour chaque site où le produit peut être trouvé.

    summary_set

    Une collection de champs de métadonnées communément utilisée pour afficher les informations lors d'une expérience basée sur une recherche de résultats.

    thumb

    Taille plus petite d'affichage ; mis à taille pour des résultats en grille. Les images vignettes sont plus petites que les aperçus et maquettes, et ont une dimension de 170 pixels sur le côté le plus long.

    title

    Indique le titre du produit.

    uri_oembed

    Identifie une URI qui peut être utilisée pour obtenir les données Embed du produit. Veuillez noter que cela sera rempli uniquement pour les produits intégrables. Voir notre documentation oEmbed pour plus d'informations.

  5. Dans le champ sort_order, sélectionnez most_popular.

    Trier les résultats de recherche vous aide à visualiser, organiser et prendre des décisions rapides concernant les photos et vidéos que vous recherchez. Best_match trie les résultats selon les images récentes pertinentes pour votre recherche ; most_popular trie les résultats selon les images les plus populaires recherchées par les clients dans le temps ; newest trie les résultats selon la date de chargement la plus récente ; oldest trie les résultats selon la date de chargement la plus ancienne.

  6. Dans le champ de requête, tapez Taylor Swift.

    Remarque : Taper une expression dans le champ requête correspond à la même fonctionnalité que taper une requête dans le champ de recherche sur le site gettyimages.fr.

    Vous souhaitez rechercher des résultats pour une autre star ? Entrez un autre nom dans ce champ.

  7. Cliquez sur le bouton Essayer.

    Le champ Réponse liste chaque image des résultats de recherche et les champs de métadonnées que vous avez sélectionnés à l'étape 4.

  8. Pour visualiser une image de vos résultats de recherche, copiez sous Réponse, la chaîne complète de l'adresse http (sans les guillemets) indiquée dans le champ uri situé en-dessous, et collez-la dans la barre d'adresse de votre navigateur par défaut.

    Dans cet exemple, la chaîne entière de l'adresse http à copier (sans les guillemets) est :

    http://cache2.asset-cache.net/gc/488755785-charles-james-beyond-fashion-costume-gettyimages.jpg?v=1&c=IWSAsset&k=2&d=X7WJLa88Cweo9HktRLaNXmrPtuUNDC4UM%2f%2bnurWTaUiLHfjnKSTc9Er6OUTQtSUrO1LhqjmtqFmEtYaCxpt12Q%3d%3d&b=ODU2

    La chaîne entière de l'adresse http affiche l'image suivante :

    Le champ Requête URL indique le code spécifique qu'une application devra envoyer à l'API afin d'obtenir ces résultats de recherche. À noter : la manière dont la requête change lorsque vous sélectionnez des expressions et des options différentes dans Swagger. Envoyer ce code à l'API a pour effet de renvoyer une réponse à une application.

    Le champ Réponse indique à quoi cette réponse ressemblera pour différents types de requêtes. Le code réponse contient toutes les informations dont un développeur a besoin afin de générer une application qui affiche une image et les informations la concernant.

Recherchez des images de stars à intégrer et visualisez les résultats de recherche.

  1. Ouvrez Swagger.
  2. Dans la colonne de gauche, cliquez sur Rechercher puis cliquez sur GET /v3/search/images/editorial.

  3. Dans le champ Clé-Api, entrez votre Clé Api embarquée.

    4. Comment obtenir une Clé Api embarquée ?

  4. Dans le panneau champs, sélectionnez uri_oembed.

    Ce champ de métadonnée fournira l'URI pouvant être utilisée pour embarquer une image sur votre site.

    Conseil : Vous pouvez également sélectionner une ou plusieurs options de métadonnées (par exemple, légende, maquette, titre, et artiste) à afficher avec les résultats de recherche.
