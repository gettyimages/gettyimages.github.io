Formas en que las empresas pueden usar la API de Getty Images

¿Administras un portal web que publica historias sobre personajes famosos? ¿Quieres hacer búsquedas y mostrar las fotografías más recientes de las celebridades en tu aplicación?

Estos procedimientos, con la herramienta Swagger de nuestra interfaz de programación de aplicaciones (API) con base web, serán el punto de partida del proceso de llamada de la API de Getty Images que solicita fotografías de la plataforma de Getty Images para exhibirlas en tu aplicación.

Este artículo contiene situaciones relacionadas con el flujo de trabajo que proporcionan ideas sobre cómo las empresas pueden usar la API de Getty Images.

  • Haz búsquedas de personajes famosos y construye una galería de imágenes
  • Haz búsquedas de imágenes insertables (embed) de personajes famosos y muestra los resultados de las búsquedas

Antes de empezar

Cerciórate de tener una clave para la API.

Para desarrolladores que crean aplicaciones con la API de Getty Images: Para asegurarte de que los resultados de búsqueda muestren correctamente los materiales a los que tu aplicación tiene acceso, además de tu clave de API, obtén un código de autorización para usar con estos flujos de trabajo.

Haz búsquedas de imágenes de personajes famosos y mira los resultados de las búsquedas

  1. Abrir Swagger.
  2. En la columna de la izquierda, haz clic en Search (buscar) y luego en GET /v3/buscar/imágenes/editorial.

  3. En el recuadro API-Key (clave de la API), introduce tu clave Sandbox Api.

    4. ¿Cómo obtengo una clave Sandbox Api?

  4. Debajo del tablero de fields (campos), selecciona una o más opciones de metadatos (por ejemplo: caption (leyenda), comp. (imagen para boceto), title (título) y artist (artista)) para que sean mostradas junto con los resultados de la búsqueda.

    Campo

    Descripción

    allowed_use

    Comunica la información sobre la licencia de uso a los clientes que hacen compras de imágenes. La mayoría de los clientes de la API acceden a las imágenes a través de sus contratos y no tienen que consultar este campo.

    artist

    Identifica al creador del material.

    asset_family

    Especifica el filtro de la familia de materiales que debe aplicarse a los resultados de la búsqueda. Los valores posibles son:

    • Creative
    • Editorial

    call_for_image

    Indica que debes llamar al representante de ventas de Getty Images para obtener acceso a los archivos de alta resolución de esta imagen. Esta imagen no está disponible para descarga online.

    caption

    Una breve descripción del material.

    collection_code

    Un código de tres letras que identifica la colección.

    collection_id

    Identifica la colección a la que pertenece el material.

    collection_name

    Identifica el nombre de la colección a la que pertenece el material.

    color_type

    Indica si el material es a color o en blanco y negro. Los valores posibles son:

    • Color
    • Black/White

    comp

    Tamaño grande de visualización; tamaño ajustado al montaje del diseño. Las imágenes para boceto son más grandes que las vistas previas y las imágenes en miniatura, pero su tamaño no es completo; en promedio, tienen alrededor de 600 píxeles en el lado más largo.

    copyright

    Identifica los derechos de autor del material.

    date_created

    Identifica la fecha de creación del material.

    detail_set

    Una colección de campos de metadatos usada habitualmente para mostrar información en los resultados de una búsqueda basada en la experiencia.

    editorial_segments

    Indica la categoría editorial a la cual pertenece el material, si la imagen forma parte de la familia de imágenes editoriales. Los valores posibles son:

    • News
    • Sports
    • Entertainment
    • Fashion
    • Celebrity portraiture
    • Royals
    • Premium photojournalism
    • Video

    event_ids

    Especifica el ID de un evento asociado con el material. Solo es relevante para las imágenes editoriales.

    graphical_style

    Indica el estilo gráfico del archivo. Los valores posibles son:

    • Photograph
    • Illustration

    id

    Una cadena numérica única que identifica el archivo.

    keywords

    Identifica el tipo de palabra clave. Los valores posibles son:

    • Unknown
    • Specific people
    • Ethnicity
    • Age
    • Gender
    • Number of people
    • Concept
    • Location
    • Viewpoint
    • Image technique
    • Composition
    • Color
    • Additional info
    • Candidate term
    • Fashion phrases
    • Entertainment

    largest_downloads

    Devuelve enlaces de descarga hipermedia para el tamaño más grande.

    license_model

    Identifica el tipo de licencia del material. Los valores posibles son:

    • RoyaltyFree
    • RightsManaged

    max_dimensions

    Dimensiones en píxeles para el archivo más grande que está disponible.

    orientation

    Especifica la orientación de los materiales que se muestran en los resultados de la búsqueda. Los valores posibles son:

    • Horizontal
    • Vertical
    • PanoramicHorizontal
    • PanoramicVertical
    • Square

    people

    Especifica los nombres de las personas que se muestran en los materiales de los resultados de la búsqueda.

    preview

    Tamaño mediano de visualización; tamaño ajustado para vistas previas emergentes. Las vistas previas son más grandes que las imágenes en miniatura pero más pequeñas que las imágenes para boceto, y tienen alrededor de 270 píxeles en el lado más largo.

    product_types

    Identifica la oferta de productos del cliente que autoriza el acceso al material. Los valores posibles son:

    • easyaccess
    • editorialsubscription
    • premiumaccess
    • royaltyfreesubscription
    • imagepack

    referral_destinations

    Contiene una entrada de "destino de la referencia" para cada uno de los sitios donde puede ser encontrado el material.

    summary_set

    Una colección de campos de metadatos usada habitualmente para mostrar información en los resultados de una búsqueda basada en la experiencia.

    thumb

    El tamaño más pequeño de visualización; el tamaño se ajusta a la cuadrícula de resultados. Las imágenes en miniatura son más pequeñas que las vistas previas y que las imágenes para boceto, y tienen una dimensión de 170 píxeles en el lado más largo.

    title

    Indica el título del material.

    uri_oembed

    Identifica una URI que puede utilizarse para obtener datos oEmbed para el material. Ten presente que solamente se completará con materiales insertables. Consulta nuestra documentación oEmbed para obtener más información.

  5. En el recuadro sort_order (ordenar por), selecciona most_popular (más popular).

    Ordenar los resultados de la búsqueda ayuda a visualizar, organizar y tomar decisiones eficaces con rapidez acerca de las fotografías y vídeos que buscas. Best_match ordena los resultados según la relevancia de las imágenes recientes para la búsqueda; most_popular ordena los resultados según la popularidad de las imágenes buscadas por los clientes a lo largo del tiempo; Newest ordena los resultados según la fecha más actual de carga; Oldest ordena los resultados según la fecha menos actual de carga.

  6. En el recuadro "Phrase (frase)", escribe Taylor Swift.

    Nota: Escribir una frase en el recuadro Phrase (frase) tiene la misma funcionalidad que escribir una frase en la caja de búsqueda en el sitio web gettyimages.es.

    ¿Quieres ver los resultados de búsqueda de otro personaje famoso? Sencillamente introduce un nombre distinto en este recuadro.

  7. Haz clic en el botón Try it out (probar).

    El recuadro Response body (cuerpo de la respuesta) presenta cada imagen de los resultados de la búsqueda y los campos de metadatos que seleccionaste en el paso 4.

  8. Para ver una imagen de los resultados de la búsqueda, debajo de Response body (cuerpo de la respuesta), copia toda la cadena de la dirección HTTP (no incluyas las comillas) que aparece en el campo uri que se encuentra a continuación y pégala en la barra de direcciones de tu navegador predeterminado.

    En este ejemplo, la cadena completa de la dirección HTTP que se debe copiar (recuerda no incluir las comillas) es:

    http://cache2.asset-cache.net/gc/488755785-charles-james-beyond-fashion-costume-gettyimages.jpg?v=1&c=IWSAsset&k=2&d=X7WJLa88Cweo9HktRLaNXmrPtuUNDC4UM%2f%2bnurWTaUiLHfjnKSTc9Er6OUTQtSUrO1LhqjmtqFmEtYaCxpt12Q%3d%3d&b=ODU2

    La cadena de la dirección HTTP muestra la siguiente imagen:

    El recuadro Request URL (solicitar URL)muestra el código específico que necesita enviar una aplicación a la API para obtener resultados como estos. Observa cómo la solicitud cambia cuando seleccionas frases y opciones distintas en Swagger. El envío de este código a la API ocasiona la devolución de una respuesta a una aplicación.

    El recuadro Response body (cuerpo de la respuesta) muestra cómo sería esta respuesta para diferentes tipos de solicitudes. El código de la respuesta contiene toda la información que el desarrollador necesita para construir una aplicación que muestre la imagen y la información de esta.

Haz búsquedas de imágenes insertables (embed) de personajes famosos y muestra los resultados de las búsquedas

  3. En el recuadro API-Key (clave de la API), introduce tu clave Api para insertar imágenes.

    4. ¿Cómo consigo una clave Api para insertar imágenes?

  4. En el tablero de fields (campos), selecciona uri_oembed.

    Este campo de metadatos proporcionará la URI que puede utilizarse para insertar una imagen en tu sitio.

    Sugerencia: También puedes seleccionar una o más opciones de metadatos (por ejemplo: caption (leyenda), comp. (imagen para boceto), title (título) y artist (artista)) para que sean mostradas junto con los resultados de la búsqueda.
