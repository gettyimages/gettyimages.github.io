PREGUNTAS FRECUENTES: Uso de OAuth 2.0 para acceder a las API de Getty Images

Las API de Getty Images requieren el uso del protocolo OAuth 2.0 para recuperar los códigos de autorización y autenticación. Estos códigos se utilizan para:

  • Identificar quién eres (autenticación)
  • Identificar a qué contenido puedes acceder o para cuál tienes licencia (autorización)

Si bien es posible hacer algunas solicitudes en la API con solo una clave API (por ejemplo, los extremos de búsqueda /v3/search/images y /v3/images no requieren un código de autorización), no lo recomendamos porque la respuesta puede no ser del todo exacta. Por ejemplo, las solicitudes de búsqueda hechas con solo una clave entregarán materiales que no podrás utilizar, porque los materiales están fuera del ámbito de la licencia concedida para tu uso.

Preguntas de autorización más frecuentes

P: ¿Cómo puedo asegurar que mis resultados de búsqueda contengan solo imágenes o vídeos para los que tengo una licencia de uso?

R: Incluye siempre un código en el encabezado de la solicitud de búsqueda, imágenes y vídeos. El código identifica quién eres y cuáles son los materiales para los que tienes una licencia de uso y así el motor de búsqueda solo entregará los materiales de esa colección de contenido.

P: Cuando realizo una solicitud de descarga, aparece el error "no autorizado". ¿Por qué?

R: Una de las razones por las que podrías estar recibiendo el error "no autorizado" es porque la imagen o el vídeo que estás tratando de descargar no está dentro del ámbito de tu acuerdo de licencia. Otra razón es que se requiere un código para todas las solicitudes de descarga, y puede que tu solicitud no tenga el código en el encabezado de autorización HTTP.

P: ¿Vence el código?

R: Sí. Los códigos generados por Client Credentials (Credenciales del cliente) y Resource Owner (Dueño del recurso) están activos durante 30 minutos.

P: ¿Cómo le permito a los usuarios iniciar sesión con su nombre de usuario de Getty Images?

R: El flujo de Implicit Grant (Concesión implícita) de OAuth 2.0 ha sido implementado para este propósito. Consulta Flujos de trabajo con autorización para obtener más información acerca de la Implicit Grant (Concesión implícita).

Flujos de trabajo con autorización

La API de Getty Images usa el protocolo OAuth 2.0 para otorgar a las aplicaciones cliente acceso a la funcionalidad de la API. La aplicación cliente usará OAuth 2.0 para solicitar un código de acceso al servidor de autorización de Getty Images, extraer un código de la respuesta y luego enviarlo a la API de Getty Images a la que deseas acceder, por ejemplo, para buscar y descargar imágenes. La API de Getty Images es compatible con los siguientes tres flujos de OAuth 2.0:

  • Client Credentials (Credenciales del cliente)
  • Resource Owner (Dueño del recurso)
  • Implicit Grant (Concesión implícita)

Flujos OAuth 2.0 compatibles con la API de Getty Images

Client Credentials (Credenciales del cliente)

El flujo de Client Credentials (Credenciales del cliente) es para aplicaciones del cliente que no tendrán usuarios individuales. Una aplicación sandbox, la cual está destinada al desarrollo de pruebas y no requiere un acuerdo de licencia, solo puede utilizar el flujo de Client Credentials (Credenciales del cliente).

Si tu empresa tiene un acuerdo de licencia con Getty Images que otorga acceso a imágenes específicas, el uso del flujo de Client Credentials (Credenciales del cliente) en las solicitudes de búsqueda te dará resultados que no reflejan con exactitud el contenido al que tienes acceso, y no podrás descargar archivos. Para obtener resultados de búsqueda y descargar los materiales que se encuentran dentro del alcance de tu acuerdo de licencia, tu clave debe estar configurada como un código con autorización completa cuando uses las Client Credentials (Credenciales del cliente). Para obtener más información sobre una configuración en particular, ponte en contacto con un gerente de cuenta de Getty Images.

El flujo de Client Credentials (Credenciales del cliente) te otorga un código de acceso de 30 minutos de duración. Una vez que el código ha vencido, debes solicitar otro. La mayoría de las aplicaciones recupera un nuevo código antes de que venza el actual.

Cómo obtener el código de Credenciales del Cliente
  1. Abre la herramienta API de Getty Images en la web.
  2. En la columna izquierda, haz clic en OAuth2, y luego clic en POST /oauth2/token – Client Credentials (Credenciales del cliente).
  3. En el recuadro client_id, ingresa tu clave API de sandbox.

    ¿Cómo obtengo una clave API de sandbox?

  4. En el recuadro client_secret, ingresa tu código secreto API.

    ¿Dónde puedo encontrar mi código secreto API?

    Abre el sitio API de Getty Images, y en la barra de navegación superior de la derecha, haz clic en Sign In (Iniciar sesión). Ingresa tu nombre de usuario y contraseña, haz clic en Sign In (Iniciar sesión), y luego ve a la página My account (Mi cuenta).

  5. Haz clic en Get Token (Obtener código).

    Notas

    • Utiliza este código para llamar los extremos de la API.
    • El flujo de Client Credentials (Credenciales del cliente) te otorga un código de 30 minutos de duración. Una vez que el código ha vencido, debes solicitar otro.
Resource Owner (Dueño del recurso)
Cómo obtener el código de concesión de Resource Owner (Dueño del recurso)
  1. Abre la herramienta API de Getty Images en la web.
  2. En la columna de la izquierda, haz clic en OAuth2, y luego clic en POST /oauth2/token – Resource Owner Grant (Concesión de dueño del recurso).

  3. En el recuadro client_id, ingresa tu clave de API.

    4. ¿Cómo obtengo una clave API de sandbox?

  4. En el recuadro client_secret, ingresa tu código secreto API.

    ¿Dónde puedo encontrar mi código secreto API?

    Abre el sitio API de Getty Images, y en la barra de navegación superior de la derecha, haz clic en Sign In (Iniciar sesión). Ingresa tu nombre de usuario y contraseña, haz clic en Sign In (Iniciar sesión), y luego ve a la página My account (Mi cuenta).

  5. Introduce tu nombre de usuario en el recuadro username.
  6. En el recuadro password, ingresa tu contraseña.
  7. Haz clic en Get Token (Obtener código).
Cómo obtener el Código de actualización
  1. Abre la herramienta API de Getty Images en la web.
  2. En la columna de la izquierda, haz clic en OAuth2, y luego clic en POST /oauth2/token – Refresh Token (Código de actualización).

  3. En el recuadro client_id, ingresa tu clave de API.

    4. ¿Cómo obtengo una clave API de sandbox?

  4. En el recuadro client_secret, ingresa tu código secreto API.

    ¿Dónde puedo encontrar mi código secreto API?

    Abre el sitio API de Getty Images, y en la barra de navegación superior de la derecha, haz clic en Sign In (Iniciar sesión). Ingresa tu nombre de usuario y contraseña, haz clic en Sign In (Iniciar sesión), y luego ve a la página My account (Mi cuenta).

  5. En el recuadro refresh_token, ingresa el valor de refresh_token que recibiste como respuesta a tu solicitud inicial de código de Resource Owner (Dueño del recurso).
Implicit Grant (Concesión implícita)

En este flujo, la aplicación cliente solicita a los usuarios que inicien sesión con su nombre de usuario y contraseña de Getty Images. Comúnmente utilizado por aplicaciones cuyos usuarios son también clientes de Getty Images (CMS, DAM, plug-ins, etc.), este flujo permite a la aplicación cliente devolver resultados de búsqueda relevantes de los materiales que clasifican en el ámbito del Acuerdo de licencia de Getty Images con el usuario.

Los usuarios son dirigidos a la página de inicio de Getty Images para introducir el nombre de usuario y contraseña, y luego son redirigidos a la aplicación. El código generado a través de este flujo es válido por un año.

Una vez que el código de acceso para un recurso dado ha vencido, un nuevo código de acceso debe ser recuperado para acceder a ese recurso. El flujo de Implicit Grant (Concesión implícita) no permite el código de actualización de acceso. Los nuevos códigos de acceso deben ser recuperados a través del flujo de Implicit Grant (Concesión implícita). Ten en cuenta que si cambias la contraseña, el código ya no será válido.

  1. La aplicación cliente llama al extremo de OAuth2 (por ejemplo, https://api.gettyimages.com/oauth2/auth/) y pasa la siguiente información:
    • Clave de API / ID del cliente
    • El URI de redirección que se ha registrado en la API de Getty Images (pueden agregarse parámetros que no están registrados)
    • Tipo de respuesta del código
    • Estado (opcional)
  2. La aplicación cliente redirige a la página de inicio de Getty Images, cuya ubicación se proporcionó en la respuesta al paso 1.
  3. El usuario final inicia sesión con su nombre de usuario y contraseña de Getty Images o Thinkstock y hace clic en Autorizar.
  4. La API verifica las credenciales del cliente y del usuario y luego redirige a la aplicación cliente con un código de acceso de larga duración.
