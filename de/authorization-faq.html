Häufig gestellte Fragen: Zugriff auf die Getty Images APIs mithilfe OAuth 2.0

Für den Zugriff auf die Getty Images APIs ist das OAuth 2.0-Protokoll für den Abruf von Autorisierungs- und Authentifizierungstoken erforderlich. Diese Token dienen folgenden Zwecken:

  • Zur Identifizierung Ihrer Person (Authentifizierung)
  • Zur Feststellung, auf welche Inhalte Sie Zugriff oder für welche Inhalte Sie eine Lizenz haben (Autorisierung)

Während es durchaus möglich ist, bestimmte API-Abrufe mit nur einem API-Schlüssel zu tätigen (die Endpunkte /v3/search/images und /v3/images erfordern beispielsweise keine Autorisierungstoken), raten wir davon ab, da Fehlerfreiheit und Genauigkeit nicht gewährleistet werden können. Zum Beispiel liefern Suchanfragen mit nur einem Schlüssel auch Treffer, die Sie nicht nutzen können, weil sie außerhalb des Verwendungsrahmens Ihrer Lizenz liegen.

Häufig gestellte Fragen zur Autorisierung

Frage: Wie kann ich sicherstellen, dass meine Suche nur Bilder oder Videos liefert, für die ich eine Lizenz habe?

Antwort: Geben Sie in der Suchmaske für Bilder und Videos stets ein Token an. Das Token ermittelt, wer Sie sind und für welche Dateien Sie eine Lizenz haben, sodass die Suche nur entsprechenden Content liefert.

Frage: Bei einem Downloadversuch erhalte ich die Fehlermeldung „Sie haben keine Zugriffsrechte“. Warum?

Antwort: Ein Grund für die Fehlermeldung wegen fehlender Zugriffsrechte könnte sein, dass das Bild oder Video, das Sie herunterzuladen versuchen, nicht von Ihrem Lizenzvertrag abgedeckt wird. Ein anderer möglicher Grund wäre, dass für alle Downloads ein Token erforderlich ist und bei Ihrer Anfrage kein Token in der HTTP-Autorisierungszeile angegeben wurde.

Frage: Kann ein Token ablaufen?

Antwort: Ja. Token, die von Client Credentials und Resource Owner generiert werden, haben eine Gültigkeitsdauer von 30 Minuten.

Frage: Wie können Nutzer sich mit ihrem Getty Images-Benutzernamen anmelden?

Antwort: Für diesen Zweck gibt es den Workflow OAuth 2.0 Implicit Grant. Weitere Informationen zu Implicit Grant finden Sie in den Autorisierungsworkflows.

Autorisierungsworkflows

Die Getty Images API greift auf das OAuth 2.0-Protokoll zurück, um Clientapplikationen Zugriff auf die API-Funktionalitäten zu gewähren. Ihre Clientapplikation greift auf das OAuth 2.0 zurück, um ein Zugriffstoken beim Getty Images Authorization Server anzufordern, ein Token aus der Antwort zu extrahieren und das Token anschließend zurück an die Getty Images API, auf die Sie zugreifen möchten, zu senden, etwa zwecks Suche und Download von Bildern. Die Getty Images API unterstützt die folgenden drei OAuth 2.0 Workflows:

  • Client Credentials
  • Resource Owner
  • Implicit Grant

Von der Getty Images API unterstützte OAuth 2.0 Workflows

Client Credentials

Der Workflow „Client Credentials“ ist für Clientanwendungen ohne individuelle Nutzer vorgesehen. Eine Sandbox-Anwendung, die für die Entwicklung von Testversionen vorgesehen ist und keine Lizenzvereinbarung erfordert, kann nur auf den Workflow „Client Credentials“ zugreifen.

Wenn Ihr Unternehmen eine Lizenzvereinbarung mit Getty Images abgeschlossen hat, der Zugriff auf bestimmte Bilder gewährt, dann liefert Ihre Suchanfrage bei Verwendung des „Client Credentials“-Workflows nicht exakt den Content, auf den Sie Zugriff haben, und ein Download der Dateien wird nicht möglich sein. Um Suchergebnisse und Dateien zu erhalten, die im Rahmen Ihrer Lizenzvereinbarung heruntergeladen werden können, muss Ihr Schlüssel als vollständig autorisiertes Token konfiguriert sein, wenn Sie Client Credentials verwenden. Um weitere Informationen zu einer bestimmten Konfiguration zu erhalten, wenden Sie sich an einen Getty Images Account Manager.

Der Client Credentials Workflow gewährt ein Zugriffstoken mit einer Gültigkeitsdauer von 30 Minuten. Nach Ablauf eines Token müssen Sie ein neues anfordern. Die meisten Anwendungen rufen ein neues Token ab, bevor das alte abläuft.

Token über Client Credentials abrufen
  1. Öffnen Sie das webbasierte Getty Images API Tool.
  2. Klicken Sie in der linken Spalte auf OAuth2 und anschließend auf POST /oauth2/token – Client Credentials.
  3. Geben Sie im Feld client_id Ihren Sandbox API-Schlüssel ein.

    Wie bekomme ich einen Sandbox API-Schlüssel?

  4. Geben Sie im Feld client_secret Ihren geheimen API-Clientschlüssel ein.

    Wo finde ich meinen geheimen API-Clientschlüssel?

    Öffnen Sie die Getty Images API Seite und klicken Sie oben rechts in der Navigationsleiste auf Anmelden. Geben Sie Benutzernamen und Passwort ein, klicken Sie auf Anmelden und gehen Sie dann zu „Mein Account“.

  5. Klicken Sie auf Token abrufen.

    Hinweise

    • Der Client Credentials Workflow gewährt ein Zugriffstoken mit einer Gültigkeitsdauer von 30 Minuten. Nach Ablauf eines Token müssen Sie ein neues anfordern.

Implicit Grant

Bei diesem Workflow fordert die Clientanwendung die Benutzer auf, sich mit ihrem Getty Images Benutzernamen und Passwort anzumelden. Dieser Workflow wird für gewöhnlich von Applikationen verwendet, deren Benutzer auch Kunden von Getty Images sind (CMSs, DAMs, Plug-ins etc.) und ermöglicht es der Clientapplikation, bei der Suche Dateien zu liefern, die im Rahmen der entsprechenden Getty Images-Lizenzvereinbarung abgedeckt werden.

Die Nutzer werden zunächst zur Getty Images Anmeldeseite weitergeleitet, wo sie sich mit Benutzernamen und Passwort anmelden, und anschließend zurück zur Applikation zurückgeleitet. Das Token, das bei diesem Workflow generiert wird, ist ein Jahr lang gültig.

Sobald ein Token für eine bestimmte Ressource abgelaufen ist, muss ein neues Zugriffstoken für den Zugriff auf diese Ressource angefordert werden. Der Workflow „Implicit Grant“ unterstützt die Tokenaktualisierung nicht. Neue Zugriffstoken müssen über den Implicit Grant Workflow angefordert werden. Das Token verliert seine Gültigkeit, wenn das Passwort geändert wird.

  1. Die Clientanwendung ruft den OAuth2-Endpunkt auf (z. B. https://api.gettyimages.com/oauth2/auth/) und gibt folgende Informationen weiter:
    • API-Schlüssel / Client-ID
    • Weiterleitungs-URI, der bei der Getty Images API registriert wurde (nicht registrierte Parameter können hinzugefügt werden)
    • Antworttyp des Token
    • Status (optional)
  2. Die Clientanwendung leitet den Benutzer zur Getty Images Anmeldeseite weiter, deren elektronische Adresse in der Antwort zu Schritt 1 bereitgestellt wird.
  3. Der Benutzer meldet sich mit seinem Getty Images oder Thinkstock Benutzernamen und Passwort an und klickt anschließend auf „Autorisieren“.
  4. Die API verifiziert die Client- und Benutzerdaten und leitet den Benutzer anschließend an die Clientanwendung mit einem langlebigen Zugriffstoken weiter.
