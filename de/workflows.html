APIs: Business Scenarios

Here are a few workflow scenarios that provide insight into ways a business can use the Getty Images API.

Search for personalities and build a gallery of images

You manage a web portal that publishes stories about celebrity sightings. You want to display the latest photos of celebrities in your application.

These procedures, using our interactive API tool, will step you through the process of initiating a Getty Images API call that will request photos from the Getty Images platform for display in your application.

Before You Begin

  • Make sure you have an API Key.
  • Make sure you have an API authorization token.

Search for images of personalities and view the search results

  1. Open Swagger
  2. In the left column, click Search, and then click GET /v3/search/images/editorial.

  3. In the Api-Key box, enter your Sandbox Api-key.
    • How do I get a Sandbox Api-key?
  4. Under fields panel, select one or more metadata options (for example, caption, comp, title, and artist) to be displayed with the search results.

    Field

    Description

    allowed_use

    Communicates licensing information to customers who make one-off image purchases. Most API customers access images through their agreements and do not need to consult this field.

    artist

    Identifies the creator of the asset.

    asset_family

    Specifies the asset family filter to apply to the search results. The possible values are:

    • Creative
    • Editorial

    call_for_image

    Indicates that you must call your Getty Images sales rep to obtain access to high res files for this image. This image is not available for online download.

    caption

    A brief description of the asset.

    collection_code

    A three-letter code identifying the collection.

    collection_id

    Identifies the collection to which the asset belongs.

    collection_name

    Identifies the name of the collection to which the asset belongs.

    color_type

    Indicates if the asset is color, or black and white. The possible values are:

    • Color
    • Black/White

    comp

    Large display size; sized for layout composition. Comp images are larger than preview and thumbnail, but not full sized, averaging about 600 pixels on the longest side.

    copyright

    Identifies asset copyright.

    date_created

    Identifies the creation date of the asset.

    detail_set

    A collection of metadata fields commonly used for displaying information in a search results based experience.

    editorial_segments

    Indicates the editorial category to which the asset belongs, if the image is in the editorial image family. Possible values are:

    • News
    • Sports
    • Entertainment
    • Fashion
    • Celebrity portraiture
    • Royals
    • Premium photojournalism
    • Video

    event_ids

    Specifies the ID of an event associated with the asset. Only relevant for Editorial images.

    graphical_style

    Indicates the graphic style of the asset. The possible values are:

    • Photograph
    • Illustration

    id

    A unique numerical string that identifies the asset.

    keywords

    Identifies the type of keyword. The possible values are:

    • Unknown
    • Specific people
    • Ethnicity
    • Age
    • Gender
    • Number of people
    • Concept
    • Location
    • Viewpoint
    • Image technique
    • Composition
    • Color
    • Additional info
    • Candidate term
    • Fashion phrases
    • Entertainment

    largest_downloads

    Returns hypermedia download links to the largest size.

    license_model

    Identifies a licensing model for the asset. The possible values are:

    • RoyaltyFree
    • RightsManaged

    max_dimensions

    Pixel dimensions of the largest available file.

    orientation

    Specifies orientation of assets returned in search results. The possible values are:

    • Horizontal
    • Vertical
    • PanoramicHorizontal
    • PanoramicVertical
    • Square

    people

    Specifies the names of people in assets returned in search results.

    preview

    Medium display size; sized for popup previews. Preview images are larger than thumbnail but smaller than comp, and are about 270 px on the longest side.

    product_types

    Identifies a customer’s product offering that authorizes access to an asset. The possible values are:

    • easyaccess
    • editorialsubscription
    • premiumaccess
    • royaltyfreesubscription
    • imagepack

    referral_destinations

    Contains a ReferralDestination entry for each of the sites on which the asset can be found.

    summary_set

    A collection of metadata fields commonly used for displaying information in a search results based experience.

    thumb

    Smallest display size; sized for a results grid. Thumbnail images are smaller than preview and comp, and have a dimension of 170 pixels on the longest side.

    title

    Indicates the title of the asset.

    uri_oembed

    Identifies a URI that can be used to get oEmbed data for the asset. Please note that this will only be populated for embeddable assets. See our oEmbed documentation for more information.

  5. In the sort_order box, select most_popular.

    Sorting search results helps you quickly visualize, organize, and make effective decisions about photos and videos youâ€™re searching for. Best_match sorts results by recent images relevant to your search; most_popular sorts results by the most popular images searched by customers over time; newest sorts results by the most current upload date; oldest sorts results by the least current upload date.

  6. In the phrase box, type Taylor Swift.

    Note: Typing a phrase in the phrase box performs the same functionality as typing a phrase in the search box on the gettyimages.com website.

    Want search results for a different personality? Simply enter a different name in this box.

  7. Click Try it out.

    The Response Body box lists each image of search results and the metadata fields you selected in step 4.

  8. To view an image from your search results, under Response Body, copy the entire HTTP address string (do not include the quotes) shown in the uri field below, and paste it in the address bar of your default browser.
