So können Unternehmen die Getty Images API nutzen

Sie verwalten ein Webportal, das Artikel über Prominente veröffentlicht. Sie möchten die neuesten Fotos von Prominenten suchen und in Ihrer Applikation anzeigen.

Im Folgenden beschreiben wir Schritt für Schritt, wie Sie mit unserem webbasierten API-Tool Swagger eine API-Anfrage an Getty Images versenden, um Fotos von der Getty Images-Plattform für die Nutzung in Ihrer Applikation anzufordern.

Dieser Artikel stellt typische Workflow-Szenarien vor, die Einblicke in die verschiedenen Verwendungsmöglichkeiten der Getty Images API für Unternehmen geben.

  • Suche nach Prominenten und Erstellen einer Bildgalerie
  • Suche nach einbettbaren Bildern von Prominenten und Anzeigen der Suchergebnisse

Erster Schritt

Stellen Sie sicher, dass Sie über einen API-Schlüssel verfügen.

Für Entwickler, die Applikationen mit der Getty Images API erstellen: Um sicherzustellen, dass Ihre Suche nur Treffer liefert, auf die Ihre Applikation zugreifen kann, benötigen Sie zusätzlich zum API-Schlüssel ein Autorisierungstoken.

Suche nach Bildern von Prominenten und Anzeigen der Suchergebnisse

  1. Öffnen Sie Swagger.
  2. Klicken Sie in der linken Spalte auf Suche (Search) und dann auf GET /v3/search/images/editorial.

  3. Geben Sie im Feld API-Schlüssel (API Key) Ihren Sandbox API-Schlüssel ein.

    4. Wie bekomme ich einen Sandbox API-Schlüssel?

  4. Wählen Sie im Steuerfeld Felder (Fields) eine oder mehrere Metadatenoptionen aus (beispielsweise Bildunterschrift (Caption), Layoutbild (Comp), Titel (Title) und Künstler (Artist)), die mit den Suchergebnissen angezeigt werden sollen.

    Feld/Halbbild

    Beschreibung

    allowed_use

    Übermittelt Kunden beim Erwerb von Einzelbildern die zugehörigen Lizenzinformationen. Die meisten API-Kunden greifen im Rahmen entsprechender Lizenzvereinbarungen auf Bilder zu und benötigen die Informationen dieses Felds nicht.

    artist

    Identifiziert den Urheber der Datei.

    asset_family

    Legt fest, welcher Familienfilter auf die Suchergebnisse anzuwenden ist. Mögliche Optionen:

    • Creative
    • Editorial

    call_for_image

    Zeigt an, dass Sie Ihren Getty Images Kundenberater anrufen müssen, um eine hochauflösende Datei des Bildes zu erhalten. Dieses Bild kann nicht online heruntergeladen werden.

    caption

    Eine kurze Beschreibung der Datei.

    collection_code

    Ein dreistelliger Code zur Identifizierung der Kollektion.

    collection_id

    Zeigt an, aus welcher Kollektion die Datei stammt.

    collection_name

    Zeigt den Namen der Kollektion an, aus der die Datei stammt.

    color_type

    Zeigt an, ob die Datei in Farbe oder Schwarzweiß gehalten ist. Mögliche Optionen:

    • Color
    • Black/White

    comp

    Große Abbildungsgröße, geeignet für die Layoutgestaltung. Layoutbilder sind größer als Vorschau- und Miniaturansichtsbilder, jedoch nicht so groß wie Bilder in Vollgröße. Die längste Seite weist durchschnittlich 600 Pixel auf.

    copyright

    Identifiziert das Urheberrecht der Datei.

    date_created

    Identifiziert das Erstellungsdatum der Datei.

    detail_set

    Eine Sammlung aus Metadatenfeldern, die häufig zur Informationsanzeige in suchergebnisbasierten Umfeldern verwendet wird.

    editorial_segments

    Zeigt die Editorial-Kategorie an, zu der die Datei gehört, falls es sich um eine Editorial-Datei handelt. Mögliche Optionen:

    • News
    • Sports
    • Entertainment
    • Fashion
    • Celebrity portraiture
    • Royals
    • Premium photojournalism
    • Video

    event_ids

    Spezifiziert die Kennnummer eines mit der Datei verbundenen Ereignisses. Nur für Editorial-Bilder relevant.

    graphical_style

    Spezifiziert den Dateityp. Mögliche Optionen:

    • Photograph
    • Illustration

    id

    Ein individueller numerischer String zur Identifizierung der Datei.

    keywords

    Identifiziert die Art des Suchbegriffs. Mögliche Optionen:

    • Unknown
    • Specific people
    • Ethnicity
    • Age
    • Gender
    • Number of people
    • Concept
    • Location
    • Viewpoint
    • Image technique
    • Composition
    • Color
    • Additional info
    • Candidate term
    • Fashion phrases
    • Entertainment

    largest_downloads

    Gibt Hypermedia-Download-Links zur größtmöglichen Dateigröße aus.

    license_model

    Identifiziert ein Lizenzmodell für die Datei. Mögliche Optionen:

    • RoyaltyFree
    • RightsManaged

    max_dimensions

    Pixelmaße der größten verfügbaren Datei.

    orientation

    Spezifiziert die Ausrichtung der gefundenen Dateien. Mögliche Optionen:

    • Horizontal
    • Vertical
    • PanoramicHorizontal
    • PanoramicVertical
    • Square

    people

    Spezifiziert die Namen der Menschen in den gefundenen Dateien.

    preview

    Mittlere Abbildungsgröße, passend für Popup-Vorschauen. Vorschaubilder sind größer als Miniaturansichts- und kleiner als Layoutbilder. Die längste Seite weist durchschnittlich 270 Pixel auf.

    product_types

    Identifiziert das Produktangebot eines Kunden, durch das der Zugriff auf eine Datei genehmigt wird. Mögliche Optionen:

    • easyaccess
    • editorialsubscription
    • premiumaccess
    • royaltyfreesubscription
    • imagepack

    referral_destinations

    Enthält eine ReferralDestination-Eingabe für jede Seite, auf der die Datei verfügbar ist.

    summary_set

    Eine Sammlung aus Metadatenfeldern, die häufig zur Informationsanzeige in suchergebnisbasierten Umfeldern verwendet wird.

    thumb

    Kleinste Abbildungsgröße, passend für Ergebnisraster. Thumbnails (Miniaturansichten) sind kleiner als Vorschau- und Layoutbilder. Die längste Seite weist 170 Pixel auf.

    title

    Zeigt den Titel der Datei an.

    uri_oembed

    Identifiziert einen URI, mit dem Sie oEmbed-Daten für die Datei erhalten. Hinweis: Diese Adresse wird nur für einbettbare Dateien bereitgestellt. Weitere Informationen siehe unsere oEmbed-Dokumentation.

  5. Wählen Sie im Feld sort_order die Option most_popular aus.

    Das Sortieren der Suchergebnisse ermöglicht es Ihnen, effektiver zu arbeiten, Bilder und Videos schneller zu sichten und zu verwalten. Mit Best_match werden die relevanten Suchergebnisse in absteigender Reihenfolge nach Datum sortiert, mit most_popular die beliebtesten von Kunden gesuchten Bilder angezeigt, mit newest zuletzt hochgeladene Ergebnisse dargestellt und mit oldest die ältesten.

  6. Geben Sie im Feld Suchbegriffe (Phrase) den Text Taylor Swift ein.

    Anmerkung: Die Funktion des Felds Suchbegriffe (Phrase) entspricht der des Suchfelds auf der Website gettyimages.de.

    Sie möchten nach einer anderen prominenten Person suchen? Geben Sie einfach einen anderen Namen in das Suchfeld ein.

  7. Klicken Sie auf die Schaltfläche Ausprobieren (Try it out).

    Im Feld Response-Body (Response Body) werden die Bilddaten der Suchergebnisse und der in Schritt 4 gewählten Metadatenfelder aufgelistet.

  8. Um ein Bild aus Ihren Suchergebnissen anzuzeigen, kopieren Sie unter Response-Body (Response Body) den gesamten HTTP-Address-String (ohne Anführungszeichen) aus dem Feld uri und fügen ihn in die Adresszeile Ihres Standardbrowsers ein.

    In unserem Beispiel lautet der zu kopierende gesamte HTTP-Address-String (ohne Anführungszeichen) wie folgt:

    http://cache2.asset-cache.net/gc/488755785-charles-james-beyond-fashion-costume-gettyimages.jpg?v=1&c=IWSAsset&k=2&d=X7WJLa88Cweo9HktRLaNXmrPtuUNDC4UM%2f%2bnurWTaUiLHfjnKSTc9Er6OUTQtSUrO1LhqjmtqFmEtYaCxpt12Q%3d%3d&b=ODU2

    Dieser HTTP-Address-String führt zum folgenden Bild:

    Das Feld Anfrage-URL (Request URL) enthält den Code, den eine Applikation an die API senden muss, um passende Suchergebnisse zu erhalten. Beachten Sie die Änderungen in der Anfrage, wenn Sie in Swagger andere Suchbegriffe und Optionen auswählen. Nach Senden dieses Codes an die API erhält die Applikation eine Antwort.

    Im Feld Response-Body (Response Body) wird angezeigt, wie diese Antwort für unterschiedliche Anfragen aussieht. Der Antwort-Code enthält alle notwendigen Informationen für Entwickler, um eine Applikation zu erstellen, die das Bild und die zugehörigen Informationen anzeigt.

Suche nach einbettbaren Bildern von Prominenten und Anzeigen der Suchergebnisse

  1. Öffnen Sie Swagger.
  2. Klicken Sie in der linken Spalte auf Suche (Search) und dann auf GET/v3/search/images/editorial.

  3. Geben Sie im Feld API-Schlüssel (API Key) Ihren Einbettungs-API-Schlüssel ein.

    4. Wie bekomme ich einen Einbettungs-API-Schlüssel?

  4. Wählen Sie im Steuerfeld Felder (Fields) die Option uri_oembed aus.

    Dieses Metadatenfeld liefert den URI, den Sie zur Einbettung eines Bildes auf Ihrer Website benötigen.

    Tipp: Sie können zusätzlich eine oder mehrere Metadatenoptionen auswählen (beispielsweise Bildunterschrift (Caption), Layoutbild (Comp), Titel (Title) und Künstler (Artist)), die mit den Suchergebnissen angezeigt werden sollen.
